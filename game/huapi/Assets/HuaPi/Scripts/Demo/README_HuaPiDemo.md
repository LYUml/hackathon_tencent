# 《画皮》两天关卡原型说明

进入 Unity Play 后，`HuaPiDemoGameController` 会自动创建运行时 Canvas，并启动一条可玩的完整原型流程。

## 当前实现流程

1. 主菜单
2. 第一天标题：第一张皮
3. 道具陈列室搜证
4. 沈青衣 / 阿喜 / 周岱 / 薛万山对话调查
5. 排练室搜证
6. 推理判定：证明“历史重演是人为复刻”
7. 观皮：用三条证据揭开沈青衣的第一张皮
8. 第二天标题：封闭后台小屋
9. 封闭小屋搜证
10. 雅座搜证与无脸人对话
11. 最终指认：锁定权限、旧案、封门和牵线
12. 结局台词：现实，是有人逼它重演

## 当前 UI 状态

当前 UI 是代码生成的临时实装版，用来先跑完玩法和关卡节奏。正式美术完成后，优先替换：

- 背景 Sprite
- 角色立绘 Sprite
- 话题按钮样式
- 证据条样式
- 五心血条图标
- 观皮黑雾 / 揭示效果
- Toast / 章节标题卡
- 最终指认界面

## 关键玩法

- 点击热点获得线索。
- 点击角色进入话题菜单。
- 特定线索会解锁追问话题。
- 背包用条状证据展示，模仿弹丸论破的证据选择节奏。
- 判定或观皮选错扣 1 心，5 心归零进入失败界面。
- 目前脚本用 `AssetDatabase.LoadAssetAtPath<Sprite>` 在编辑器中直接加载素材。若要打正式包，需要把素材迁移到 Resources、Addressables 或正式 prefab 引用。

## 后续接入建议

先保留 `HuaPiDemoGameController` 的流程和数据，只把 UI 创建部分替换为正式 Prefab。等美术资源稳定后，再把流程数据逐步拆到已有的 `ExplorationPanel`、`DialoguePanel`、`CluePanel`、`ObserveSkinPanel`。
