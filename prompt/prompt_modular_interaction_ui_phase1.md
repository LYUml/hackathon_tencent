# Prompt：模块化游戏内交互 UI 第一阶段

你是腾讯 WorkBuddy。请基于当前项目文件夹中的所有已有信息继续开发《画皮》的 Unity UI，但本轮不要一次性制作所有页面。本轮只做第一阶段：**模块化场景探索系统**，并用一个后台化妆间 Demo 验证它可以运行。

请先读取并理解：

- `prompt/` 文件夹中的游戏概念和 UI 架构 prompt。
- `ui/plan/` 中已有的 UI 方案，尤其是 V3、V4 的“无边界沉浸式 UI”方向。
- `game/huapi/Assets/HuaPi/Scripts/` 中现有的 `UIManager`、`PanelBase`、`UIStack` 和各 Panel 脚本。
- `game/huapi/Assets/HuaPi/Prefabs/` 中已有的 UI Prefab。
- `pic/背景图/` 中已有的场景图。
- `pic/人物立绘/` 中已有的人物立绘。
- `game/huapi/Assets/Fonts/` 中已有的字体资源，包括 `hakidame SDF`。

## 本轮目标

请不要推倒重来，也不要只写设计说明。本轮目标是：

1. 在现有 Unity UGUI 架构基础上，新增或完善一个可复用的场景探索 UI 系统。
2. 让这个系统可以更换背景图、配置调查热点、配置可点击人物、显示当前目标、弹出获得线索提示。
3. 用一组固定 Demo 数据验证系统：后台化妆间探索、点击旦角人物入口、点击镜台热点、获得“镜台下的旧戏票”线索。
4. 为下一阶段的对话页、线索页、设置页预留接口，但本轮不要把这些页面全部做完。

核心体验链路：

`场景探索页 → 点击人物入口或调查热点 → 获得线索提示 → 预留进入对话/线索页接口`

## 风格要求

请继续保持《画皮》当前 UI 方向：

- 16:9 横屏，基准分辨率 1920 x 1080。
- Unity UGUI，不使用 UI Toolkit。
- 2D 漫画 / 插画风格。
- 暗黑中式戏曲恐怖氛围。
- 无边界沉浸式 UI。
- 不要厚重矩形框、明显卡片框、多边形框、系统窗口感。
- UI 信息应像从场景、黑雾、旧纸、戏票、皮影、墨迹、幕布中自然浮现。
- 标题、短提示、恐怖强调文字可以使用 `hakidame SDF`。
- 正文、目标说明、线索描述要使用更易读的 TextMeshPro 中文字体。
- 鼠标 Hover 时 UI 绝对不能消失，不能把 Alpha、Scale 或 SetActive 设置到不可见状态。

## 必须实现的模块

请优先实现或完善以下可复用模块。命名可以根据现有代码风格微调，但职责必须清楚。

### 1. `ExplorationPanel`

场景探索页主面板。

职责：

- 管理当前场景背景。
- 管理调查热点。
- 管理可点击人物入口。
- 管理当前地点和当前目标显示。
- 管理获得线索提示。
- 提供进入对话页、线索页、设置/暂停页的接口。

要求：

- 继承或遵循现有 `PanelBase` 逻辑。
- 注册到现有 `UIManager` 或现有面板管理系统。
- 不要写死单一场景；应支持通过配置切换场景内容。

### 2. `SceneBackgroundView`

全屏场景背景显示模块。

职责：

- 显示 `pic/背景图/` 中的场景图。
- 适配 16:9，铺满屏幕但不拉伸变形。
- 支持压暗、黑雾叠加或轻微暗场渐变。

要求：

- 使用 UGUI `Image` 或适合 UI 的图片组件。
- 背景需要完整沉浸，不要像普通 UI 背板。
- 背景图应导入 Unity 后设置为 Sprite 或 UI 可用资源。

### 3. `SceneHotspotItem`

场景调查热点模块。

职责：

- 表示场景中一个可调查物件。
- 支持 Hover 显示物件名。
- 支持点击后触发获得线索、提示文本或后续事件。

要求：

- 默认状态低调，不能破坏场景图。
- Hover 状态可以使用暗金描边、墨迹浮现、轻微发光、物件名浮现。
- 不要显示现代按钮框。
- 不要在 Hover 时隐藏 UI。

### 4. `SceneCharacterEntry`

可点击人物入口模块。

职责：

- 显示或引用当前场景中的人物立绘。
- 支持点击人物进入后续对话模块。
- 本轮只需要预留接口和基础点击反馈，不需要完整对话系统。

要求：

- 使用 `pic/人物立绘/旦.png` 作为 Demo 人物。
- 人物立绘放在画面左侧或右侧，不能遮挡关键调查区。
- Hover 时可以让人物轮廓微亮、阴影加深或显示角色名。

### 5. `ObjectiveNoteView`

当前目标提示模块。

职责：

- 显示当前短期目标。
- 在探索页中低调常驻或短暂浮现。

Demo 文案：

- 当前地点：`后台化妆间`
- 当前目标：`调查镜台附近的异常，并试着与旦角交谈。`

视觉要求：

- 像墨迹、旧纸条或场景中的字，不要像系统任务栏。
- 字体必须清楚可读。

### 6. `ClueToastView`

获得线索提示模块。

职责：

- 当玩家点击正确热点后，显示获得线索提示。
- 提示之后可以保留线索状态，供后续线索页使用。

Demo 线索：

- 线索名：`镜台下的旧戏票`
- 来源地点：`后台化妆间`
- 来源物件：`镜台`
- 描述：`一张被灰尘和胭脂沾污的旧戏票，角落残留着“薛家戏班”的字样。票根边缘有轻微烧痕。`

视觉要求：

- 像旧纸条、暗红印章、墨迹浮现或戏票滑入。
- 不要做现代弹窗。
- 文字字号要足够大，1920 x 1080 下清楚可读。

## 验收标准

完成后应满足：

- Unity 中能出现一个 16:9 的后台化妆间探索 Demo。
- 背景图来自 `pic/背景图/`，不是纯色占位。
- 人物立绘来自 `pic/人物立绘/`，优先显示旦角。
- 场景中至少有一个镜台热点。
- 鼠标 Hover 镜台热点时有清楚但不突兀的反馈。
- 点击镜台后能显示获得线索提示：`镜台下的旧戏票`。
- 当前地点和目标能显示。
- 有进入对话、线索页、暂停页的接口或占位入口。
- 代码和 Prefab 结构是模块化的，后续可以替换背景、热点、人物和线索。
- Hover 不会导致任何 UI 消失。

核心目标：

先把《画皮》的游戏内探索 UI 从“页面概念”推进到“可复用、可拼装、可运行的模块化场景探索系统”。后台化妆间只是第一组示例数据，后续所有场景都要能按同样逻辑配置出来。
