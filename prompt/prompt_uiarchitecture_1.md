# Prompt：UI 风格设计与 Unity 架构

你是腾讯 WorkBuddy，负责为游戏开发比赛项目《画皮》设计一套可在 Unity 中落地的 UI 风格方案与 UGUI 架构方案。请先理解游戏的核心气质：这是一款以中国传统戏曲文化、画皮传说、脸谱、皮影戏和民国旧戏班为灵感的叙事推理解谜游戏。玩家白天调查线索，夜晚进入“观皮”阶段，通过证据驱散角色隐藏画像上的黑雾，逐步揭开脸谱之下的真实身份、谎言与罪恶。

你的输出不应该只是好看的 UI 概念图描述，也不应该只是抽象代码架构。请同时给出“适合这个游戏的 UI 风格设计”和“可以交给 Unity 开发者实现的 UGUI 架构设计”。

## 设计目标

UI 必须服务于《画皮》的核心体验：

- 让玩家清楚知道当前处于白天调查还是夜晚观皮。
- 让玩家能快速查看线索、角色档案和当前目标。
- 让玩家在对话、调查、推理之间自然切换。
- 让“黑雾消散、真相显现”的过程成为最有记忆点的 UI 反馈。
- 让界面具有中国戏曲、旧戏班、民国悬疑和水墨恐怖的气质，但不能牺牲可读性和操作效率。

UI 的第一优先级是清晰可用，第二优先级是氛围表达，第三优先级才是复杂装饰。

## 默认技术栈

请默认使用 Unity UGUI，而不是 UI Toolkit。

可使用的核心 Unity UI 技术包括：

- Canvas
- Canvas Scaler
- Graphic Raycaster
- EventSystem
- Image
- Button
- TextMeshPro
- ScrollRect
- Toggle
- Slider
- CanvasGroup
- Animator
- RectTransform
- Prefab

请围绕 UGUI 的实际开发方式进行架构设计，避免提出只适合网页或 Figma 的方案。

## UI 视觉风格

整体 UI 气质应当是：暗红、黑墨、旧纸、戏台、脸谱、皮影、压抑、悬疑、克制。

推荐视觉元素：

- 暗红色作为主要强调色，关联戏台帷幕、血色记忆和旧案阴影。
- 黑色与深灰作为基础背景色，关联水墨、夜晚和隐藏真相。
- 米黄、旧纸色或褪色白作为文字和信息面板底色，增强可读性。
- 金色或暗铜色作为少量高亮，用于重要线索、当前选择和关键按钮。
- 京剧脸谱纹样可用于标题、角色档案边框、按钮装饰或章节标记。
- 皮影戏剪影可用于转场、空状态、档案装饰和夜晚观皮背景。
- 黑雾、水墨扩散、纸张破损和火烧痕迹可用于揭露反馈。

请避免：

- 过亮、过现代、过科幻的 UI。
- 大面积高饱和红色导致视觉疲劳。
- 为了装饰牺牲文字可读性。
- 所有界面都堆满脸谱和纹样，导致重点不清。

## 字体与文字层级

请设计一套适合中文游戏的文字层级。

建议包含：

- 标题字体：可带有书法、牌匾或戏曲海报感，但必须保证可读。
- 正文字体：清晰、稳定，适合长段中文阅读。
- 数字和系统提示：简洁醒目，适合快速扫描。

请明确说明：

- 主标题、副标题、正文、线索名、按钮文字、系统提示分别使用什么字号范围。
- 重要信息如何通过颜色、字号、粗细或图标区分。
- 对话文本应如何保证长句阅读舒适。
- 浏览器端构建时如何避免字体缺失问题。

## 核心页面设计

请为以下页面分别设计功能、布局、交互和视觉重点。

### 1. 主菜单

主菜单应当立即传达《画皮》的氛围。

需要包含：

- 开始游戏
- 继续游戏
- 设置
- 退出或返回

视觉建议：

- 背景可以是空戏台、半开的红帷幕、悬挂脸谱、旧剧院入口或黑雾中的角色剪影。
- 菜单按钮应当像旧戏票、木牌、纸签或戏班告示，而不是现代扁平按钮。
- 当前选中项可以通过暗金描边、轻微墨迹扩散或烛光闪动表现。

### 2. 白天探索 HUD

白天 HUD 应当轻量，不能遮挡场景调查。

需要显示：

- 当前时间阶段：白天
- 当前地点
- 当前短期目标
- 线索入口
- 角色档案入口
- 暂停入口

交互要求：

- 可调查物件应有低调但清楚的高亮反馈。
- 鼠标悬停或选中时可以出现物件名称。
- 获得线索时应有短暂提示，不打断探索节奏。

### 3. 对话界面

对话界面要支持叙事、调查和矛盾感。

需要包含：

- 角色名
- 角色立绘或剪影区域
- 对话文本
- 对话选项
- 已获得线索提示
- 可疑语句或关键词强调

视觉建议：

- 对话框可以参考戏台字幕牌、旧纸卷、剧本纸页或后台记录册。
- 对话选项不要太花，重点保证玩家能判断选择。
- 关键矛盾句可以用轻微墨迹、暗红标记或下划线提示，但不要过度游戏化。

### 4. 线索 / 证据背包

证据界面是推理体验的核心之一。

每条线索至少显示：

- 线索名称
- 简短描述
- 来源地点或来源人物
- 关联角色或事件
- 是否已用于观皮

布局建议：

- 左侧为线索列表或分类。
- 右侧为线索详情。
- 可按地点、人物、时间或是否已使用筛选。
- 比赛版本可以先实现最小版本：线索列表 + 详情面板 + 已使用状态。

### 5. 角色档案

角色档案需要体现“表面身份”和“隐藏真相”的冲突。

每个角色档案建议包含：

- 角色姓名或称呼
- 表面身份
- 当前脸谱或舞台身份
- 玩家已知信息
- 可疑点
- 已关联线索
- 隐藏画像揭露进度

视觉建议：

- 档案可以像侦探笔记、戏班名册、旧照片册或角色行当表。
- 角色脸谱可以作为表面身份标记。
- 隐藏画像区域应保留给夜晚观皮系统使用。

### 6. 夜晚“观皮”界面

观皮界面是整个 UI 的核心记忆点。

需要包含：

- 当前观皮角色
- 被黑雾覆盖的隐藏 CG 或隐藏画像
- 可选择的线索列表
- 已揭露区域
- 推理反馈
- 返回或确认按钮

交互要求：

- 玩家选择线索后，将线索应用到角色隐藏画像或档案节点。
- 正确线索触发黑雾消散，并露出隐藏画像的一部分。
- 错误线索给出明确反馈，但不要强惩罚。
- 揭露后应解锁新文字、新疑点或新目标。

视觉建议：

- 背景可使用夜晚戏台、黑色幕布、烛光、皮影屏风或水墨暗场。
- 黑雾应像水墨扩散一样覆盖画像。
- 驱散黑雾时可使用墨迹退散、纸张燃开、灯光照出或面具剥落的动效。

### 7. 黑雾揭露反馈

黑雾揭露是本项目最重要的 UI 表现。

请设计：

- 普通线索获得反馈
- 正确线索匹配反馈
- 黑雾消散动效
- 新信息解锁反馈
- 关键秘密揭露反馈

反馈原则：

- 普通线索反馈要轻，不打断节奏。
- 观皮成功反馈要明显，有仪式感。
- 关键秘密揭露可以短暂停顿、压低环境音、放大画面细节。
- 不要使用过于现代的弹窗和成就提示破坏氛围。

### 8. 暂停 / 设置界面

设置界面可以简单，但需要完整。

建议包含：

- 继续游戏
- 音量设置
- 文字速度
- 画面亮度
- 返回主菜单
- 退出游戏

比赛版本中可以先实现基础设置界面，但结构应方便后续扩展。

## Unity UGUI 架构要求

请设计一套适合比赛 Demo 的 UGUI 架构，目标是清晰、可维护、易扩展，而不是过度工程化。

### Canvas 分层

建议至少包含以下 UI 层级：

- 背景层：菜单背景、观皮背景、转场底图。
- 游戏 HUD 层：白天探索 HUD、当前目标、地点信息、快捷入口。
- 常规面板层：线索背包、角色档案、对话界面。
- 弹窗层：确认框、线索获得提示、错误提示。
- 揭露 / 过场层：黑雾消散、关键秘密揭露、章节转场。
- 系统菜单层：暂停、设置、主菜单覆盖层。

请说明每一层的 Canvas 或 Panel 组织方式，以及层级顺序如何避免遮挡错误。

### UI 管理结构

请设计以下基础结构：

- UIManager：负责打开、关闭、切换 UI 面板。
- PanelBase：所有主要面板的基础类，统一处理初始化、显示、隐藏、刷新。
- UIPanelType：用于标识不同面板类型。
- 面板栈：用于支持返回上一个界面，例如从线索详情返回线索列表，从角色档案返回探索 HUD。
- 弹窗管理：用于显示确认框、提示框和线索获得提示。

请说明：

- 哪些面板是常驻的。
- 哪些面板是按需打开的。
- 哪些面板会暂停游戏输入。
- 哪些面板允许背景继续显示。

### Prefab 拆分

请提出一套可复用 UI Prefab 方案。

至少包括：

- 通用按钮 Prefab
- 主菜单按钮 Prefab
- 对话框 Prefab
- 对话选项 Prefab
- 线索卡片 Prefab
- 线索详情 Prefab
- 角色档案卡 Prefab
- 角色隐藏画像区域 Prefab
- 黑雾遮罩 Prefab
- 观皮线索槽 Prefab
- 线索获得提示 Prefab
- 确认弹窗 Prefab

每个 Prefab 请说明它的职责、主要子节点和需要暴露给脚本的字段。

### 数据驱动要求

UI 应尽量由数据驱动，方便比赛期间快速改内容。

请建议数据结构或 ScriptableObject 类型，用于配置：

- 线索数据
- 角色档案数据
- 对话数据
- 观皮揭露节点
- UI 文本
- 颜色和基础样式

每条线索建议包含：

- 线索 ID
- 线索名称
- 描述
- 来源地点
- 来源人物
- 关联角色
- 是否已获得
- 是否已用于观皮

每个角色档案建议包含：

- 角色 ID
- 角色名
- 表面身份
- 脸谱类型
- 已知信息
- 可疑点
- 隐藏画像资源
- 揭露进度

每个观皮揭露节点建议包含：

- 节点 ID
- 关联角色 ID
- 所需线索 ID
- 揭露区域
- 揭露后文本
- 是否触发新目标

## 交互与输入

默认平台为 PC 和浏览器端 Unity 构建。

请优先支持鼠标点击交互：

- 点击调查物件
- 点击对话选项
- 点击打开线索背包
- 点击查看角色档案
- 点击或拖拽线索到观皮区域
- 点击返回、关闭、确认

可以补充键盘快捷键，但不要让快捷键成为必需操作。

建议快捷键：

- Esc：打开暂停或返回上一级
- Tab：打开线索界面
- C：打开角色档案
- Space 或鼠标左键：推进对话

请说明如何避免 UI 输入与场景调查输入冲突。

## 动效与反馈

动效应当克制、有主题感，不能让 UI 变慢。

建议动效：

- 面板打开：轻微淡入、纸张展开、墨迹显现。
- 面板关闭：淡出、墨迹收回、纸页合上。
- 线索获得：小幅震动、暗金描边、短暂墨迹闪现。
- 按钮悬停：暗红或暗金边缘亮起。
- 观皮成功：黑雾向外退散，隐藏画像局部显现。
- 观皮失败：线索轻微暗下或墨迹回流。

请说明哪些动效必须实现，哪些可以作为后续 polish。

## MVP 优先实现顺序

请按比赛 Demo 的优先级设计开发顺序。

建议顺序：

1. 建立基础 UGUI 框架：Canvas 分层、UIManager、PanelBase、基础按钮。
2. 实现白天探索 HUD：当前地点、目标、线索入口、角色档案入口。
3. 实现对话界面：角色名、文本、选项、推进对话。
4. 实现线索系统 UI：线索列表、线索详情、线索获得提示。
5. 实现角色档案 UI：角色基础信息、可疑点、已关联线索。
6. 实现夜晚观皮 UI：隐藏画像、黑雾遮罩、线索选择、结果反馈。
7. 实现黑雾揭露动效：至少完成一个可用的局部揭露效果。
8. 实现暂停和设置界面。
9. 统一视觉样式、字体、颜色和动效节奏。
