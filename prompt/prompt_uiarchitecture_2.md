# Prompt：UI 风格设计与 Unity 架构第二版

你是腾讯 WorkBuddy，负责为游戏开发比赛项目《画皮》设计一套可在 Unity 中落地的 UI 风格方案与 UGUI 架构方案。请基于第一版 UI 方向继续深化，但第二版必须重点吸收以下新要求：

1. 游戏画面与 UI 设计必须以 16:9 横屏比例为基准。
2. 整体风格需要比第一版更恐怖、更暗、更压抑。
3. 美术风格必须明确为 2D 漫画 / 插画风格，而不是写实 3D、半写实 3D 或现代扁平 UI。
4. UI 风格必须参考 `character` 文件夹中的人物设计图，包括青衣、小生、文丑、净行·武净等角色图。

请输出一份可以指导 Unity UGUI 实现的完整 UI 风格与架构方案。你的方案需要同时解决视觉风格、页面流程、Canvas 分层、Prefab 拆分、数据驱动、输入交互和 MVP 实现优先级。

## 项目背景

《画皮》是一款以中国传统戏曲文化、画皮传说、脸谱、皮影戏和民国旧戏班为灵感的叙事推理解谜游戏。玩家扮演一名年轻侦探，伪装成戏班学徒潜入古老戏班调查旧剧院火灾与失踪案。

核心玩法是：

- 白天调查：探索场景、对话、收集线索。
- 夜晚观皮：整理证据、匹配角色档案、驱散隐藏画像上的黑雾。
- 逐步揭露：通过黑雾消散看到人物脸谱之下的真实身份、伤痕、谎言与罪恶。

UI 的任务不是单纯装饰画面，而是帮助玩家完成调查、整理证据、阅读人物、进入观皮，并让每次揭露都具有恐怖感和仪式感。

## 第二版核心设计方向

第二版 UI 的关键词是：

- 16:9 横屏
- 深色恐怖
- 2D 漫画插画
- 旧戏班
- 暗红黑墨
- 脸谱异化
- 纸张腐坏
- 皮影剪影
- 亡魂戏子
- 民国悬疑

整体体验应当更接近“阴冷的 2D 恐怖漫画游戏界面”，而不是普通视觉小说界面。玩家看到 UI 时，应当立刻感到：这是一个关于戏台、亡魂、脸谱和隐藏真相的恐怖推理游戏。

## 16:9 画面比例要求

所有 UI 设计必须以 16:9 横屏为基础。

请默认设计基准分辨率：

- 1920 x 1080

Unity UGUI 设置建议：

- Canvas Scaler 使用 `Scale With Screen Size`。
- Reference Resolution 使用 `1920 x 1080`。
- Screen Match Mode 使用 `Match Width Or Height`。
- Match 建议从 `0.5` 开始，具体由 WorkBuddy 根据布局说明微调。

布局设计必须考虑：

- 主要信息集中在 16:9 安全区域内。
- 不要把关键按钮放在过于贴边的位置。
- 对话框、证据栏、角色档案和观皮界面都必须适配 1920 x 1080。
- 浏览器端运行时，即使窗口缩放，也不能出现按钮被裁切、文字溢出或关键图像被遮挡。

建议安全边距：

- 左右至少保留 80 px。
- 上下至少保留 60 px。
- 对话和证据面板不要超过屏幕高度的 40% 到 50%，避免遮挡 2D 角色和场景。

请在输出中明确说明每个核心页面的 16:9 布局方案。

## 更恐怖、更暗的视觉方向

第二版必须比第一版更暗、更恐怖、更有压迫感。

推荐主色：

- 近黑色：用于背景、遮罩、深夜界面。
- 暗红色：用于危险、血色记忆、戏台帷幕、关键线索。
- 脏灰蓝：用于亡魂、寒冷、旧墙和夜间空气。
- 暗铜金：用于极少量高亮、传统纹样和可交互边框。
- 旧纸黄：用于文字承载区，但饱和度要低，不能显得温暖明亮。

请避免：

- 大面积明亮红色。
- 干净的米白色。
- 现代蓝色科技感 UI。
- 过多金色造成华丽感。
- 过于可爱、轻松或卡通化的恐怖。

恐怖感应来自：

- 低明度背景。
- 高对比白脸 / 黑影。
- 局部暗红提示。
- 破损纸边、血污、烧痕、墨迹。
- 角色脸谱和真实面孔之间的不适感。
- 黑雾遮挡与局部显现。
- 静默、停顿、微弱闪烁和画面压暗。

UI 不应变成“全黑看不清”。请保证可读性：正文文字、按钮状态、当前目标和线索名称必须清楚。

## 2D 漫画 / 插画风格要求

本游戏的 UI 与角色呈现必须统一为 2D 漫画 / 插画风格。

请参考 `character` 文件夹中的人物设计图，提炼以下美术特征：

- 角色为竖幅 2D 立绘。
- 使用清晰的黑色轮廓线和细密线稿。
- 服装有较多传统戏曲纹样和层次。
- 颜色整体偏暗、偏脏、偏旧。
- 人物脸部有强烈脸谱或白面妆感。
- 服装存在破损、血渍、毛边、污痕等亡魂化改编。
- 恐怖感来自“传统戏曲形制 + 亡魂异化”的对比。

请将这些特征迁移到 UI 设计中：

- UI 边框可以像漫画分镜框、旧纸框、戏牌框或破损档案框。
- 图标应使用 2D 手绘线稿风格，而不是现代矢量扁平图标。
- 按钮可以像撕裂的戏票、旧戏牌、纸签、木牌或破损剧本页。
- 背景装饰可以使用手绘黑影、皮影轮廓、脸谱残片、墨线和烧痕。
- 观皮界面的黑雾应像 2D 手绘墨雾，而不是写实粒子烟雾。
- 角色档案中的角色图应使用 2D 立绘或剪影，不要使用 3D 模型预览。

请明确说明 UI 中哪些元素需要采用手绘线稿，哪些元素可以保持简洁以保证可读性。

## 核心页面设计要求

请为以下页面设计 16:9 横屏布局、视觉风格、交互方式和 Unity UGUI 结构。

### 1. 主菜单

目标：

- 一进入游戏就传达更暗、更恐怖的 2D 戏曲漫画氛围。

16:9 布局建议：

- 左侧或中左区域放置游戏标题《画皮》。
- 右侧或中右区域放置菜单按钮。
- 背景使用 2D 插画：空戏台、半开的暗红帷幕、悬挂脸谱、皮影屏风、黑雾中的青衣或武净剪影。
- 底部可有极暗的旧剧院地面、火烧痕迹或散落戏票。

菜单项：

- 开始游戏
- 继续游戏
- 设置
- 退出游戏

视觉要求：

- 按钮像破损戏票、旧木牌或手写戏单。
- 选中状态使用暗红描边、墨迹扩散或低频闪烁。
- 不要使用明亮高饱和按钮。

### 2. 白天探索 HUD

目标：

- 白天仍然压抑，不要变得明亮轻松。
- HUD 必须轻量，给场景和角色立绘留出空间。

16:9 布局建议：

- 左上角显示当前地点与时间阶段。
- 右上角放置暂停按钮和系统入口。
- 右下角放置线索 / 档案快捷入口。
- 底部仅在需要时显示当前目标提示。

视觉要求：

- HUD 使用暗色半透明旧纸或墨色底板。
- 可调查物件高亮用暗铜金或灰白描边，不要用现代蓝色高亮。
- 线索获得提示从屏幕右侧或下方短暂滑入，像一张被墨迹污染的纸条。

### 3. 对话界面

目标：

- 支持中文长文本阅读，同时保持漫画式恐怖叙事感。

16:9 布局建议：

- 角色立绘占屏幕左侧或右侧 35% 到 45%。
- 对话框位于底部，占屏幕高度约 28% 到 35%。
- 对话选项浮在对话框上方或右侧，不能遮挡角色脸部。

视觉要求：

- 对话框像破旧剧本页、戏台字幕牌或烧边纸张。
- 角色名牌可以使用小戏牌样式。
- 可疑词句使用暗红点染、墨迹下划线或轻微抖动，不要使用现代荧光高亮。
- 对话推进时可有轻微纸张翻动或墨字显现效果。

### 4. 线索 / 证据背包

目标：

- 让玩家快速整理证据，同时保留侦探档案与旧戏班气质。

16:9 布局建议：

- 左侧 30% 为线索列表。
- 右侧 60% 为线索详情。
- 顶部为分类筛选：全部、人物、地点、火灾、已使用。
- 右下角保留返回按钮。

视觉要求：

- 线索卡片像破损纸片、旧照片、戏票或烧边档案。
- 已使用线索可盖上暗红印章，例如“已观皮”。
- 未获得线索不要用亮色问号，可用黑色墨块或被撕掉的空位表示。

### 5. 角色档案

目标：

- 强化“表面行当 / 隐藏真相”的冲突。

16:9 布局建议：

- 左侧显示角色 2D 立绘或脸谱剪影。
- 中间显示表面身份、行当、已知信息、可疑点。
- 右侧显示隐藏画像揭露进度与关联线索。

视觉要求：

- 档案像侦探笔记、戏班名册、旧照片册和漫画分镜的结合。
- 每个角色可有少量识别色，但整体必须保持暗色统一。
- 隐藏画像未揭露区域应被黑雾、墨块或撕裂纸遮挡。
- 角色图不要做成圆形头像，优先使用竖向卡片或半身立绘。

### 6. 夜晚“观皮”界面

目标：

- 这是整个游戏最重要的 UI 场景，必须比第一版更恐怖、更有仪式感。

16:9 布局建议：

- 中央 45% 到 55% 区域放置被黑雾覆盖的隐藏画像。
- 左侧放置角色档案摘要。
- 右侧放置可用线索列表。
- 底部放置当前推理提示、返回和确认按钮。

视觉要求：

- 背景使用极暗戏台、皮影屏风、黑色幕布或烧毁剧院。
- 画像框像旧供桌上的相框、戏班牌位、破损剧照或漫画恐怖分镜。
- 黑雾是 2D 墨雾和手绘阴影，不要做成写实烟雾粒子。
- 正确线索匹配时，黑雾向外退散或像墨水被水冲开。
- 关键揭露时，画面可以短暂压暗，只留下白脸、伤痕或脸谱裂纹。

### 7. 黑雾揭露反馈

目标：

- 让“揭开画皮”成为玩家最记得住的瞬间。

必须设计：

- 线索选中反馈
- 正确匹配反馈
- 错误匹配反馈
- 黑雾局部消散
- 关键秘密揭露
- 新目标解锁提示

反馈风格：

- 正确匹配：暗红线条连接线索与画像，墨雾被撕开。
- 错误匹配：线索卡片被黑墨吞回，发出短促低沉反馈。
- 关键揭露：画面停顿 0.5 到 1 秒，环境音压低，局部白脸或伤口被照亮。
- 新目标：不要用现代任务弹窗，用旧纸条、侦探笔记或血色印章呈现。

### 8. 暂停 / 设置界面

目标：

- 保持恐怖氛围，不要像系统软件。

需要包含：

- 继续游戏
- 音量
- 文字速度
- 亮度
- 返回主菜单
- 退出游戏

视觉要求：

- 背景压暗并保留当前场景轮廓。
- 设置面板像旧剧本夹、木牌或破损后台告示。
- Slider 和 Toggle 需要做成戏曲/旧纸风格，而不是 Unity 默认风格。

### Canvas 分层

请设计以下 Canvas 或 Panel 层级：

- BackgroundCanvas：主菜单插画背景、观皮背景、转场背景。
- WorldHUDCanvas：白天探索 HUD、地点、目标、快捷入口。
- DialogueCanvas：对话框、角色名牌、对话选项。
- NormalPanelCanvas：线索背包、角色档案、设置页。
- PopupCanvas：线索获得提示、确认框、错误反馈。
- RevealCanvas：黑雾消散、关键秘密揭露、画面压暗、恐怖闪帧。
- SystemCanvas：暂停菜单、主菜单覆盖层。

请说明：

- 每层的 Sort Order 建议。
- 哪些层常驻，哪些层按需实例化。
- 哪些层需要阻断场景输入。
- 哪些层允许背景继续可见。

### UI 管理结构

请设计以下基础结构：

- UIManager：统一打开、关闭、切换和返回 UI。

- UIStack：管理返回路径和弹窗层级。

- ObserveSkinPanel：管理夜晚观皮界面。
- RevealEffectController：管理黑雾消散和关键揭露动效。

请给出每个模块的职责边界，避免一个脚本承担所有 UI 逻辑。

## 动效与恐怖反馈

第二版动效应更恐怖，但不能拖慢操作。

必须实现的 MVP 动效：
