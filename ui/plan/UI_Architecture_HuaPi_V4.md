# 《画皮》第四版 UI 风格与 Unity UGUI 架构方案

> 版本：V4.0 纯净沉浸版
> 基准：16:9 横屏 1920×1080 | 2D 漫画/插画风格 | 无边界沉浸式 UI | Unity UGUI
> 日期：2026-06-22
> 核心目标：开始界面像一张完整的恐怖 2D 插画，UI 从场景、黑雾、旧幕布和戏曲文字中自然浮现，更沉浸、更纯净、更少界面感。

---

## 一、设计哲学：从场景本身浮现的 UI

### 核心原则

**UI 不是“放在画面上的面板”，而是“从场景本身浮现的信息”——更进一步：UI 应该是场景本身。**

在《画皮》中，玩家进入的不是一个“游戏菜单”，而是一个旧戏班、一座烧毁的剧院、一群戴着脸谱的亡魂。开始界面应该让玩家感觉：我进入的不是一个菜单，而是一张旧戏班的照片、一个被黑雾笼罩的舞台、一个被遗忘的剧场。

### 边界感消除策略（V4 强化版）

| 传统 UI 元素 | V4 处理方式 | 效果 |
|-------------|------------|------|
| 厚重矩形面板 | 完全去除，信息直接写在场景中 | 没有“面板”，只有场景和文字 |
| 明显边框和描边 | 完全去除，或用极淡线稿（opacity 0.1–0.2） | 边框几乎不可见 |
| 独立弹窗/窗口 | 用场景内元素浮现（墨迹显现、文字从暗场中渗出来） | 没有“打开了一个窗口”的感觉 |
| 矩形按钮 | 不规则形状或纯文字（无底板） | 更像场景中的文字或物品 |
| 卡片式列表 | 散落的旧纸/照片/戏票，或纯文字 | 像侦探在桌上整理证据 |
| 纯黑背景遮罩 | 半透明暗场 + 场景残像 | 背景仍有场景氛围 |
| 高对比白底 | 旧纸/墨色底色，与场景自然过渡 | 不刺眼，融入暗场 |
| 多边形 UI 框 | 完全去除，不再使用 | 没有几何装饰框 |

### 可读性保障（V4 强化）

无边界不等于看不清。保证可读性的手段：
- **文字区域**：文字背后有极暗半透明渐变（`#0a0808`，opacity 0.6–0.8），确保与任何背景对比度 ≥ 4.5:1。
- **字号**：不小于 12px（系统提示）和 15px（正文），禁止在暗场上使用小于 11px 的字。
- **颜色**：正文用 `#d8cfc0`（旧纸色），高亮用 `#c9a96e`（暗金），危险用 `#8b2020`（暗红），都是高对比色。
- **Hover/选中**：暗红渗墨、文字轻微扭曲、阴影加深、墨迹扩散、微弱闪烁或低饱和血色描边。

## 二、字体方案：hakidame（吐满了）+ 易读字体

### 字体选择

| 用途 | 字体 | 说明 | 安装方式 |
|------|------|------|----------|
| **标题/主菜单** | hakidame（吐满了） | 笔画粗、字形扭曲、部件比例失调、大小对比明显，恐怖戏曲气质 | 安装字体文件后，在 TextMeshPro 中创建 Font Asset |
| **正文/对话** | 思源黑体/Noto Sans SC | 清晰、长段中文阅读，避免阅读困难 | 内嵌 TTF，TMP 预生成 Atlas |
| **数字/系统提示** | Noto Sans SC Medium | 干净、快速扫描 | 内嵌 TTF，TMP 预生成 Atlas |
| **书法/戏曲感文字** | 站酷庆科黄油体/思源宋体 Heavy | 用于特殊场景（如章节标题、档案标签） | 内嵌 TTF，TMP 预生成 Atlas |

### hakidame 字体使用建议

- **标题“画皮”**：使用 hakidame，字号 48–56px，颜色 `#8b2020`（暗红），opacity 0.9。
- **主菜单文字**：使用 hakidame，字号 16–18px，颜色 `#c9a96e`（暗金），opacity 0.85。
- **关键恐怖提示**：使用 hakidame，字号 14–16px，颜色 `#8b2020`（暗红）。
- **正文/对话**：使用思源黑体/Noto Sans SC，字号 15–16px，颜色 `#d8cfc0`（旧纸色）。
- **系统提示**：使用 Noto Sans SC，字号 12–13px，颜色 `#6b5a4a`（墨灰）。

### TextMeshPro 配置建议

1. **hakidame Font Asset**：
   - 在 Unity 中导入 hakidame 字体文件（`.ttf` 或 `.otf`）。
   - 打开 `Window > TextMeshPro > Font Asset Creator`。
   - 选择 hakidame 字体文件，设置 `Atlas Resolution` 为 2048×2048（保证清晰度）。
   - `Character Set` 选择 `Unicode Range (Hex)`，输入常用汉字范围（如 `4E00-9FFF` 覆盖基本汉字）。
   - 点击 `Generate Font Atlas`，生成 `.asset` 文件。
   - 将生成的 Font Asset 放入 `Assets/Fonts/` 目录。

2. **思源黑体/Noto Sans SC Font Asset**：
   - 同样使用 Font Asset Creator 生成。
   - `Atlas Resolution` 建议 4096×4096（中文字符较多）。
   - `Character Set` 选择 `Unicode Range (Hex)`，输入 `4E00-9FFF`。
   - 生成后将 Font Asset 放入 `Assets/Fonts/`。

3. **Fallback 字体设置**：
   - 在 `TMP Settings`（`Window > TextMeshPro > Project Settings`）中，设置 `Default Font Asset` 为思源黑体/Noto Sans SC。
   - 为 hakidame 字体设置 Fallback 为思源黑体/Noto Sans SC（防止生僻字显示不出来）。

4. **WebGL 构建优化**：
   - 字体 Subset 裁剪：只包含游戏中实际使用的字符，减小 Atlas 大小。
   - 使用 `TMP_FontAsset` 的 `Atlas` 压缩格式（如 `RGBA 32 bit` 或 `Alpha 8 bit`）。
   - 如果包体过大，可将标题字体（hakidame）单独打包，正文字体使用系统默认字体作为 Fallback。

## 三、16:9 布局与安全区（V4 调整）

### 设计基准（不变）
- **分辨率**：1920 × 1080
- **Canvas Scaler**：`Scale With Screen Size`
- **Reference Resolution**：1920 × 1080
- **Screen Match Mode**：`Match Width Or Height`（0.5，观皮界面 0.6）

### V4 安全边距调整

| 方向 | 安全边距 | 原因 |
|------|----------|------|
| 左右 | ≥ 50 px | 比 V3 减少（60→50），信息可以更贴近边缘 |
| 上下 | ≥ 40 px | 比 V3 减少（50→40），信息可以更接近顶部和底部 |
| 对话区域 | 底部 20%–25% | 保持与 V3 一致，为场景和角色留出更多空间 |
| 线索/档案区域 | 全屏散点布局 | 不再局限于左右面板，信息可以分散在场景各处 |

### 关键变化：UI 即场景

- **开始界面**：像一张完整的恐怖 2D 插画，标题和菜单是画面中的文字，不是 UI 元素。
- **HUD 信息**：像墨迹漂浮在场景上方，没有矩形底板。
- **对话文字**：像戏台字幕从底部浮现，背景是极暗渐变而非旧纸框。
- **线索**：像散落在桌面上的旧纸和照片，点击时纸张轻微浮起。
- **观皮**：玩家直接面对黑雾中的画像，线索像漂浮的墨迹，没有“打开面板”的感觉。

## 四、色彩方案（V4 微调）

V4 保持 V3 色彩体系，但调整使用方式：**更强调透明度和渐变，减少实色块，更纯净**。

| 用途 | 色值 | V4 使用方式 |
|------|------|------------|
| **背景极深** | `#0a0808` | 全屏暗场，opacity 0.95–1.0，场景主基调 |
| **背景深** | `#1a1210` | 用于信息区域底色，opacity 0.3–0.6（更透明） |
| **旧纸** | `#d8cfc0` | 文字色，无实色背景，文字背后用极暗半透明渐变 |
| **强调暗红** | `#8b2020` | 危险、关键线索、标题，用于文字和极细线条，面积 ≤ 3% |
| **暗铜金** | `#c9a96e` | 选中态、高亮，用于文字和 hover 描边，面积 ≤ 5% |
| **脏灰蓝** | `#4a5a6a` | 亡魂冷色，用于小生角色相关，opacity 0.5–0.7 |
| **正白** | `#f0ece4` | 仅用于关键揭露时的局部高亮，平时不用 |
| **墨灰** | `#5a5a5a` | 次要文字、未激活状态 |
| **黑雾** | `#0a0808` | 纯黑，opacity 0.85–0.95，2D 手绘墨雾叠加 |

### V4 色彩使用规则
- **禁止大面积实色块**：即使是“面板”，也用半透明渐变（`#0a0808` opacity 0.4 渐变到透明）。
- **文字可读性**：文字背后必须有暗底，但暗底用渐变而非矩形。
- **暗红和暗金**：只用于文字和线条，不用于大面积填充。
- **hakidame 字体颜色**：标题用 `#8b2020`（暗红），菜单用 `#c9a96e`（暗金），hover 时用 `#8b2020`（暗红）。

## 五、开始界面设计：一张完整的恐怖 2D 插画

### 布局（16:9）

- **背景**：全屏 2D 插画——极暗戏台、红色帷幕（极淡）、黑雾（弥漫在下半部分）、悬挂脸谱（极淡）、皮影剪影（极淡）。**整个画面像一张完整的恐怖插画，不是“背景图 + UI 层”。**
- **标题“画皮”**：画面正中上方（y ≈ 30%），使用 hakidame 字体，字号 48–56px，颜色 `#8b2020`（暗红），opacity 0.9。标题像写在黑暗中的血字，或者从幕布上渗出来的字。
- **英文副标题**：标题下方（y ≈ 35%），"MASKS BEHIND MASKS"，字号 9–10px，颜色 `#5a3a2a`（墨灰），opacity 0.6。像雾中的低语。
- **菜单项**：画面正中下半部分（y ≈ 55%–80%），低于标题。使用 hakidame 字体，字号 16–18px，竖排或居中纵向排列。
  - 开始游戏（暗金，opacity 0.85）
  - 继续游戏（暗金，opacity 0.7）
  - 设置（暗金，opacity 0.7）
  - 退出（墨灰，opacity 0.55）
- **底部信息**：画面底部中央（y ≈ 95%），"薛家戏班 · 民国二十年"，字号 8–9px，颜色 `#3d1518`（极暗），opacity 0.4。像戏台上的灰尘或旧字迹。
- **亡魂剪影**：画面下半部分（y ≈ 60%–80%），极淡的皮影剪影（opacity 0.1–0.15），像雾中的影子，不抢标题和菜单焦点。

### 边界感消除（开始界面）

- ❌ **完全去除**：
  - 标题矩形底板
  - 菜单矩形按钮/卡片
  - 菜单多边形框
  - 背景与 UI 的分界线
  - 几何装饰框
  - 悬挂物的明显描边
- ✅ **保留**：
  - 标题文字（hakidame 字体，高对比）
  - 菜单文字（hakidame 字体，高对比）
  - 背景 2D 插画（戏台、帷幕、黑雾、脸谱）
  - 亡魂剪影（极淡，融入背景）
- ✅ **新增**：
  - 标题下方淡淡的墨迹渗开（手绘线稿，opacity 0.3–0.4）
  - 菜单项 hover 时的暗红渗墨/微弱闪烁
  - 黑雾弥漫在画面下半部分，自然过渡到菜单区域

### 菜单项布局细节

- **排列方式**：竖排（居中纵向排列），每个菜单项之间间距 40–50px。
- **对齐**：所有菜单项水平居中对齐（`text-anchor="middle"`）。
- **字号**："开始游戏" 略大（18px），其他略小（16px）。
- **颜色层次**：
  - 开始游戏：暗金 `#c9a96e`，opacity 0.85（最亮，引导玩家点击）
  - 继续游戏：暗金 `#c9a96e`，opacity 0.7
  - 设置：暗金 `#c9a96e`，opacity 0.7
  - 退出：墨灰 `#8a7a6e`，opacity 0.55（最暗，表示次要选项）
- **Hover 状态**：见下方“Hover 状态设计”。

## 六、Hover 状态设计：暗红渗墨、文字扭曲、微弱闪烁

### 菜单项 Hover 状态（开始界面）

| 状态 | 视觉表现 | 实现方式 | 持续时间 |
|------|----------|----------|----------|
| **默认** | 暗金文字，opacity 0.7–0.85 | `TMP_Text.color = #c9a96e` | 无 |
| **Hover** | 文字颜色变暗红 `#8b2020`，opacity 0.9，文字下方出现淡淡墨迹渗开（手绘线稿） | `TMP_Text.color` 从 `#c9a96e` 渐变到 `#8b2020`（200ms），下方 `Image`（墨迹线稿）`alpha` 从 0 到 0.4（300ms） | 200–300ms |
| **选中** | 文字颜色保持暗红，opacity 1.0，文字轻微向左偏移 1px（扭曲感），下方墨迹渗开更明显 | `TMP_Text.color = #8b2020`，`alpha = 1.0`，`RectTransform.anchoredPosition.x` 从 0 到 -1（100ms），下方墨迹线稿 `alpha` 从 0.4 到 0.6 | 100ms |
| **微弱闪烁** | 文字 opacity 在 0.85–1.0 之间微弱波动（0.3s 间隔） | `CanvasGroup.alpha` 用协程循环 0.85→1.0→0.85 | 循环 |

### 通用 Hover 状态（其他界面）

| 元素 | Hover 表现 | 实现方式 |
|------|------------|----------|
| **对话选项** | 文字颜色从 `#9c8a6e` 变 `#c9a96e`，文字下方出现淡淡墨迹线 | `TMP_Text.color` 渐变 + 下方 `Image`（墨迹线稿）`alpha` 渐变 |
| **线索** | 旧纸签轻微浮起（y 偏移 -3px，scale 1.01），暗金描边亮起 | `RectTransform.anchoredPosition.y` + `scale` + `Image.color` |
| **角色档案** | 旧照片边缘暗金描边亮起，照片轻微放大（scale 1.02） | `Image.color` + `RectTransform.scale` |
| **观皮线索** | 旧纸签暗红描边亮起，文字颜色变暗红 | `Image.color` + `TMP_Text.color` |
| **设置 Slider** | 暗铜滑块轻微放大，墨迹轨道颜色变暗金 | `RectTransform.scale` + `Image.color` |

## 七、动效设计：从场景中浮现

### 开始界面动效

1. **进入画面**：
   - 背景 2D 插画从全黑（`alpha = 1`）渐变到正常（`alpha = 0`），持续 1.5s，像舞台灯光慢慢亮起。
   - 标题“画皮”从下方缓慢浮现（`y` 从 +30 到 0，`alpha` 从 0 到 0.9），持续 1s，延迟 0.5s。
   - 菜单项依次浮现（每个延迟 0.2s，`alpha` 从 0 到 0.7–0.85），持续 0.5s。
   - 亡魂剪影从透明到半透明（`alpha` 从 0 到 0.15），持续 2s，延迟 1s。

2. **标题效果**：
   - 标题下方有淡淡的墨迹渗开动画（手绘线稿 `alpha` 从 0 到 0.4，持续 0.8s，延迟 1.5s）。
   - 标题可以轻微呼吸（`scale` 从 1 到 1.01，循环 3s），像活物。

3. **黑雾动画**：
   - 画面下半部分的黑雾轻微飘动（多个圆形 `Image` 的 `x` 和 `y` 轻微偏移，循环 5s，偏移 ±5px）。
   - 黑雾透明度轻微波动（`alpha` 从 0.7 到 0.85，循环 4s）。

### 通用动效（其他界面）

1. **文字浮现**：
   - `alpha` 从 0 到 1（300ms），`y` 从 +10 到 0（轻微上浮感）。
   - 像文字从暗场中“渗出来”。

2. **旧纸浮现**：
   - `alpha` 从 0 到 1（400ms），`scale` 从 0.95 到 1（轻微放大感），`rotation` 从 ±3° 到 0（像纸被铺平）。

3. **黑雾退散**：
   - `CanvasGroup.alpha` 从 0.95 到 0.2（800ms），配合 `scale` 从 1 到 1.1（向外扩散感）。

4. **错误墨吞回**：
   - `Image.color.a` 从 1 到 0.3（200ms），`scale` 从 1 到 0.95（收缩感），然后恢复。

## 八、其他界面调整（V4 纯净版）

### 1. 白天探索 HUD：场景中的墨迹

- 左上角：地点信息——像写在旧墙上的墨迹，只有文字，没有底板。文字背后有极暗渐变（`#0a0808`，opacity 0.5，渐变到透明）。
- 右上角：暂停按钮——像一个小皮影剪影，hover 时剪影亮起。
- 右下角：线索/档案快捷入口——像两个旧戏票小角标，贴在画面角落，点击时展开。
- 底部中央：当前目标——像从底部飘出的旧纸条，只有一行字，2 秒后自然淡入黑雾。
- 可调查物件：暗铜金虚线描边（极细，opacity 0.4），hover 时物件名称像墨迹从物件旁浮现。

### 2. 对话界面：戏台字幕与漂浮纸条

- 左侧 35%：角色 2D 立绘，无矩形框，立绘边缘用暗场渐变自然融入背景。
- 角色名：像用墨写在暗场中的小字，无底板。
- 对话文字：底部 20%–25% 区域，像戏台字幕从暗场中浮现。文字背后有极暗渐变（`#0a0808`，opacity 0.7，从底部向上渐变到透明），不是矩形旧纸框。
- 对话选项：像漂浮的旧墨迹/纸条，从对话框右侧或上方飘出，每个选项是无规则形状，无矩形卡片感。
- 已有关联线索：像角落的墨迹字，不显眼但可点击展开。

### 3. 线索 / 证据背包：散落的旧纸与侦探笔记

- 背景：暗场场景（如侦探房间的桌面、后台的化妆台），线索像散落在桌面上的旧纸、照片、戏票。
- 左侧：线索列表——不是矩形列表，而是散落的旧纸，每张纸是不规则形状，有撕裂边缘。已选中的纸略微浮起（y 偏移 -5px，scale 1.02）。
- 右侧：线索详情——像侦探展开的一张旧纸，纸是不规则形状，上面有文字和暗红印章。
- 筛选标签：像压在纸上的小墨块/标签，点击时墨块变色。
- 返回：像一个角落的皮影剪影，点击时剪影亮起。

### 4. 角色档案：摊开的侦探笔记与旧照片

- 背景：暗场桌面，像侦探在桌上摊开一本笔记和几张旧照片。
- 左侧：角色 2D 立绘——像一张竖立的旧照片，照片边缘破损，没有矩形框。
- 中间：侦探笔记——像摊开的笔记本，纸页是不规则形状，上面手写文字。表面身份、行当、可疑点像笔记中的条目。
- 右侧：隐藏画像——像另一张被黑雾覆盖的照片，放在笔记旁边。黑雾是 2D 手绘墨雾，覆盖在照片上方。
- 关联线索：像贴在笔记边缘的小纸条，每张纸条是不规则形状，有暗红印章。

### 5. 夜晚观皮界面：直接面对黑雾中的画像

- 背景：极暗烧毁剧院/戏台，背景装饰 opacity ≤ 10%，几乎纯黑。
- 中央 45%：被黑雾覆盖的隐藏画像——像挂在墙上的一张旧照片，照片本身没有明显边框，只有极淡的暗铜线稿（opacity 0.2）。黑雾是 2D 手绘墨雾，多个不规则黑色圆形叠加，覆盖在照片上方。玩家感觉像真的在看一张被黑雾遮住的照片。
- 左侧：角色信息——像墨迹浮在暗场中，只有文字，没有底板。角色名、行当、可疑点像用墨写在黑雾边缘。
- 右侧：可用线索——像漂浮的旧纸签/墨迹，每个线索是不规则形状，没有矩形卡片。
- 底部：操作提示——像暗红烛光/墨迹字，极小，不抢中央画像焦点。

### 6. 黑雾揭露反馈：从场景中直接显现

- **线索选中**：线索像被墨笔点中，暗红线条从线索位置延伸到画像区域（手绘线，不是几何直线）。
- **正确匹配**：黑雾从线索延伸点向外退散，像墨水被冲开。没有“弹窗”或“通知”，而是画像本身局部变亮，新信息像从画像中浮现。
- **错误匹配**：线索像被黑墨吞回，纸签变暗并轻微收缩（scale 0.95），没有弹窗，只有纸签本身的反馈。
- **关键揭露**：画面没有弹窗，而是中央画像局部压暗四周，只留白脸/伤痕/裂纹被照亮。新文字像从画像中浮现，不是从面板中弹出。
- **新目标**：像一张旧纸条从画面底部自然滑入，贴在场景中，2 秒后融入黑雾。

### 7. 暂停 / 设置：暗场中的微弱烛光

- 背景：全屏半透明黑底（`#0a0808`，opacity 0.8），保留当前场景轮廓，但压暗。
- 设置选项：像暗场中的微弱烛光/墨迹字，垂直排列在画面中央。每个选项没有矩形底板，只有文字和极淡的暗铜线稿（opacity 0.2）。
- Slider：像旧纸上的墨迹线，滑块是暗铜小点，不是现代矩形滑块。
- Toggle：像旧纸上的勾选框，手绘对勾，无矩形边框。
- 按钮：像暗场中的墨迹字，hover 时字亮（暗金），没有矩形底。

## 十二、Unity UGUI 实现建议（V4 纯净版）

### 1. Canvas 与透明度

- **BackgroundCanvas**：使用 `Screen Space - Overlay`，Sort Order 0。背景图是全屏 2D 插画（1920×1080 PNG）。
- **其他层**：使用 `Screen Space - Overlay`，Sort Order 10–60。所有 UI 元素的 `Image` 组件使用半透明颜色（`color.a` 0.3–0.9）。
- **关键**：使用 `CanvasGroup` 控制整层透明度，而不是单个 `Image`。

### 2. 无边框实现

- **去除 `Image` 的默认矩形**：使用不规则 Sprite（2D 插画师提供），`Image.type = Simple`。
- **边框处理**：不要给 `Image` 加 `Outline` 组件，而是使用极淡线稿 Sprite（单独一层，opacity 0.1–0.3）。
- **背景处理**：不要使用 `Image` 的实色背景，而是使用渐变 Sprite（从 `#0a0808` opacity 0.7 渐变到透明）。

### 3. 遮罩与黑雾

- **黑雾遮罩**：使用多个 `Image`（圆形 Sprite）叠加，每个 `Image.color = #0a0808`，`alpha` 0.85–0.95。
- **黑雾退散**：使用 `CanvasGroup.alpha` 或 `Image.color.a` 渐变，从 0.95 到 0.2，持续 500–800ms。
- **关键揭露压暗**：在 RevealCanvas 上添加全屏 `Image`（`#0a0808`，alpha 0.7），中央用遮罩挖洞（或用一个透明圆形区域），只保留中央画像区域可见。

### 4. 动效与无边界感

- **浮现效果**：使用 `CanvasGroup.alpha` + `RectTransform.anchoredPosition`。
  - 文字浮现：`alpha` 0→1（300ms），`y` 从 +10 到 0（轻微上浮感）。
  - 旧纸浮现：`alpha` 0→1（400ms），`scale` 从 0.95 到 1（轻微放大感），`rotation` 从 ±3° 到 0（像纸被铺平）。
- **黑雾退散**：`CanvasGroup.alpha` 从 0.95 到 0.2（800ms），配合 `scale` 从 1 到 1.1（向外扩散感）。
- **错误墨吞回**：`Image.color.a` 从 1 到 0.3（200ms），`scale` 从 1 到 0.95（收缩感），然后恢复。
- **关键揭露停顿**：全屏黑底 `alpha` 从 0 到 0.7（500ms），停顿 0.5 秒，中央画像区域 `alpha` 保持，然后黑底 `alpha` 从 0.7 到 0.3（800ms），新文字浮现。

### 5. 交互反馈

- **Hover**：暗红渗墨（`Image.color` 从 `#5a3a2a` 到 `#8b2020`，200ms），或文字轻微向左偏移 1px（`RectTransform.anchoredPosition.x` 从 0 到 -1，100ms）。
- **选中**：文字 `scale` 从 1 到 1.02（轻微放大），暗红描边亮起。
- **可点击区域**：使用 `Image` 的 `raycastTarget = true`，但 `Image.color.a` 可以很低（0.1），玩家看不到明显的点击区域，但鼠标悬停时有反馈。
- **拖拽**：观皮时拖拽线索到画像，线索像被墨笔拖曳，留下淡淡墨迹轨迹（用 `TrailRenderer` 或 `LineRenderer` 模拟）。

### 6. 场景融合技术

- **主菜单**：背景图是全屏戏台插画，菜单项是纯文字 Sprite（或 TMP_Text），用锚点固定在画面中央。
- **对话**：背景是场景图（半暗），角色立绘是左侧 Sprite，对话文字是底部半透明渐变文字。
- **线索背包**：背景是侦探房间插画，线索是散落在背景上的 Sprite，点击时放大到中央。
- **观皮**：背景是烧毁剧院插画，画像是中央 Sprite，黑雾是覆盖在画像上的多个 Sprite。

### 7. hakidame 字体配置

- **导入**：将 hakidame 字体文件（`.ttf`）拖入 Unity `Assets/Fonts/` 目录。
- **生成 Font Asset**：`Window > TextMeshPro > Font Asset Creator`，选择 hakidame 字体，设置 `Atlas Resolution` 2048×2048，`Character Set` 为 `Unicode Range (Hex)`，输入 `4E00-9FFF`。
- **使用**：在 TMP_Text 组件中，将 `Font Asset` 设置为 hakidame，用于标题和主菜单文字。
- **Fallback**：在 `TMP Settings` 中设置 Fallback 为思源黑体/Noto Sans SC，防止生僻字显示不出来。

## 十四、关键设计原则总结（V4）

> **UI 不是放在画面上的面板，而是《画皮》世界本身——从黑雾、旧幕布和戏曲文字中自然浮现出来。**

1. **场景即界面**：主菜单就是戏台，对话就是字幕，线索就是散落的旧纸，观皮就是黑雾中的画像。
2. **信息无框**：文字直接出现在场景中，背后只有极暗渐变，没有矩形底板。
3. **元素无形**：按钮像旧戏票，选项像旧纸签，线索像旧照片，所有元素都是不规则形状。
4. **黑雾即遮罩**：黑雾不是“UI特效”，而是场景本身的一部分，2D 手绘墨雾覆盖在画像上。
5. **动效即氛围**：浮现、退散、墨吞回，所有动效都像是水墨和旧纸在变化，不是数字面板的动画。
6. **可读性不妥协**：即使无边界，文字背后仍有暗底渐变，对比度 ≥ 4.5:1，字号不小于 12px。
7. **hakidame 字体**：标题和主菜单使用 hakidame 字体，增强恐怖戏曲气质，正文仍用易读字体。
