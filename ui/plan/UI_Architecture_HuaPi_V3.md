# 《画皮》第三版 UI 风格与 Unity UGUI 架构方案

> 版本：V3.0 无边界沉浸式版
> 基准：16:9 横屏 1920×1080 | 2D 漫画/插画风格 | 无边界沉浸式 UI | Unity UGUI
> 日期：2026-06-22
> 核心目标：UI 从“有边框的游戏菜单”变成“融入《画皮》世界的沉浸式恐怖戏曲界面”

---

## 一、设计哲学：无边界沉浸式 UI

### 核心原则

**UI 不是“放在画面上的面板”，而是“从场景本身浮现的信息”。**

在《画皮》中，玩家调查的是一座旧戏班、一座烧毁的剧院、一群戴着脸谱的亡魂。UI 应该让玩家感觉：这些信息本身就是戏班的一部分——是旧纸上的墨迹、是幕布上的字迹、是漂浮的皮影、是黑雾中的残像。

### 边界感消除策略

| 传统 UI 元素 | V3 处理方式 | 效果 |
|-------------|------------|------|
| 厚重矩形面板 | 用场景元素替代（幕布、旧纸、墙壁）或完全去除 | 信息直接出现在场景中 |
| 明显边框和描边 | 极淡线稿（opacity 0.2–0.4）或完全去除 | 边框不抢内容焦点 |
| 独立弹窗/窗口 | 用场景内元素浮现（纸条滑出、墨迹显现） | 没有“打开了一个窗口”的感觉 |
| 矩形按钮 | 不规则形状（撕裂纸边、墨迹块、木牌） | 更像场景中的物品 |
| 卡片式列表 | 散落的旧纸/照片/戏票 | 像侦探在桌上整理证据 |
| 纯黑背景遮罩 | 半透明暗场+场景残像 | 背景仍有场景氛围 |
| 高对比白底 | 旧纸/墨色底色，与场景自然过渡 | 不刺眼，融入暗场 |

### 可读性保障

无边界不等于看不清。保证可读性的手段：
- **文字区域**：即使无面板，文字背后仍有极暗半透明底（`#0a0808`，opacity 0.6–0.8），确保与任何背景对比度 ≥ 4.5:1。
- **字号**：不小于 12px（系统提示）和 15px（正文），禁止在暗场上使用小于 11px 的字。
- **颜色**：正文用 `#d8cfc0`（旧纸色），高亮用 `#c9a96e`（暗金），危险用 `#8b2020`（暗红），都是高对比色。
- **Hover/选中**：暗金描边或亮度提升，让玩家明确知道可交互区域。

## 二、16:9 布局与安全区（V3 调整）

### 设计基准（不变）
- **分辨率**：1920 × 1080
- **Canvas Scaler**：`Scale With Screen Size`
- **Reference Resolution**：1920 × 1080
- **Screen Match Mode**：`Match Width Or Height`（0.5，观皮界面 0.6）

### V3 安全边距调整

| 方向 | 安全边距 | 原因 |
|------|----------|------|
| 左右 | ≥ 60 px | 比 V2 减少（80→60），因为无边界 UI 可以更贴近边缘 |
| 上下 | ≥ 50 px | 比 V2 减少（60→50），信息可以更接近顶部和底部 |
| 对话区域 | 底部 20%–25% | 比 V2 减少（28%→25%），为场景和角色留出更多空间 |
| 线索/档案区域 | 全屏散点布局 | 不再局限于左右面板，信息可以分散在场景各处 |

### 关键变化：信息不再“框在面板里”

- **HUD 信息**：像墨迹漂浮在场景上方，没有矩形底板。
- **对话文字**：像戏台字幕从底部浮现，背景是极暗渐变而非旧纸框。
- **线索**：像散落在桌面上的旧纸和照片，点击时纸张轻微浮起。
- **观皮**：玩家直接面对黑雾中的画像，线索像漂浮的墨迹，没有“打开面板”的感觉。

## 三、色彩方案（V3 微调）

V3 保持 V2 色彩体系，但调整使用方式：**更强调透明度和渐变，减少实色块**。

| 用途 | 色值 | V3 使用方式 |
|------|------|------------|
| **背景极深** | `#0a0808` | 全屏暗场，opacity 0.95–1.0，场景主基调 |
| **背景深** | `#1a1210` | 用于信息区域底色，opacity 0.4–0.7（半透明），而非实色块 |
| **旧纸** | `#d8cfc0` | 文字色，无实色背景，文字背后用极暗半透明渐变 |
| **强调暗红** | `#8b2020` | 危险、关键线索，用于文字和极细线条，面积 ≤ 3% |
| **暗铜金** | `#c9a96e` | 选中态、高亮，用于文字和 hover 描边，面积 ≤ 5% |
| **脏灰蓝** | `#4a5a6a` | 亡魂冷色，用于小生角色相关，opacity 0.5–0.7 |
| **正白** | `#f0ece4` | 仅用于关键揭露时的局部高亮，平时不用 |
| **墨灰** | `#5a5a5a` | 次要文字、未激活状态 |
| **黑雾** | `#0a0808` | 纯黑，opacity 0.85–0.95，2D 手绘墨雾叠加 |

### V3 色彩使用规则
- **禁止大面积实色块**：即使是“面板”，也用半透明渐变（`#0a0808` opacity 0.6 渐变到透明）。
- **文字可读性**：文字背后必须有暗底，但暗底用渐变而非矩形。
- **暗红和暗金**：只用于文字和线条，不用于大面积填充。

## 四、无边界布局语言：各页面重新设计

### 1. 主菜单：空戏台即菜单

**布局（16:9）**：
- 背景：全屏 2D 插画——空戏台、暗红帷幕、悬挂脸谱、黑雾、亡魂剪影。没有“背景图”和“UI层”的分界，整个画面就是戏台。
- 标题：融入左侧幕布，像旧戏班招牌上的字，没有矩形背景板。
- 菜单：右侧悬挂的旧戏票/木牌，不规则形状，撕裂边缘，像真的挂在幕布旁。
- 底部：黑雾自然过渡到戏台地面，没有明显的“面板底边”。

**边界感消除**：
- ❌ 去除：标题矩形底板、菜单矩形按钮、背景与UI的分界线。
- ✅ 保留：标题文字（高对比）、菜单文字（高对比）、选中态暗金描边（让玩家知道能点击）。
- ✅ 新增：菜单项像悬挂物，有轻微摆动动画（模拟悬挂），hover 时摆动幅度略增。

### 2. 白天探索 HUD：场景中的墨迹

**布局（16:9）**：
- 左上角：地点信息——像写在旧墙上的墨迹，只有文字，没有底板。文字背后有极暗渐变（防止与场景背景冲突）。
- 右上角：暂停按钮——像一个小皮影剪影，hover 时剪影亮起。
- 右下角：线索/档案快捷入口——像两个旧戏票小角标，贴在画面角落，点击时展开。
- 底部中央：当前目标——像从底部飘出的旧纸条，只有一行字，2 秒后自然淡入黑雾。
- 可调查物件：暗铜金虚线描边（极细，opacity 0.5），hover 时物件名称像墨迹从物件旁浮现。

**边界感消除**：
- ❌ 去除：HUD 矩形底板、地点标签矩形框、快捷入口矩形按钮。
- ✅ 保留：文字本身、虚线描边（hover 时）、物件名称浮现。
- ✅ 调整：所有 HUD 元素用 `CanvasGroup.alpha` 0.7–0.9，半透明，不遮挡场景。

### 3. 对话界面：戏台字幕与漂浮纸条

**布局（16:9）**：
- 左侧 35%：角色 2D 立绘，无矩形框，立绘边缘用暗场渐变自然融入背景。
- 角色名牌：像小戏牌悬挂在角色上方，不规则形状，暗红底 + 暗金文字，无矩形边框。
- 对话文字：底部 20%–25% 区域，像戏台字幕从暗场中浮现。文字背后有极暗渐变（`#0a0808`，opacity 0.7，从底部向上渐变到透明），不是矩形旧纸框。
- 对话选项：像漂浮的旧纸签/墨迹，从对话框右侧或上方飘出，每个选项是不规则形状，无矩形卡片感。
- 已有关联线索：像角落的墨迹字，不显眼但可点击展开。

**边界感消除**：
- ❌ 去除：旧纸卷对话框矩形框、选项矩形卡片、角色名牌矩形底。
- ✅ 保留：文字可读性（暗底渐变）、选项 hover 态（暗金描边）、角色名牌（小戏牌形状）。
- ✅ 新增：文字像从暗场中“渗出来”的感觉（`alpha` 从 0 渐变到 1，带轻微 y 偏移）。

### 4. 线索 / 证据背包：散落的旧纸与侦探笔记

**布局（16:9）**：
- 背景：暗场场景（如侦探房间的桌面、后台的化妆台），线索像散落在桌面上的旧纸、照片、戏票。
- 左侧：线索列表——不是矩形列表，而是散落的旧纸，每张纸是不规则形状，有撕裂边缘。已选中的纸略微浮起（y 偏移 -5px，scale 1.02）。
- 右侧：线索详情——像侦探展开的一张旧纸，纸是不规则形状，上面有文字和暗红印章。
- 筛选标签：像压在纸上的小墨块/标签，点击时墨块变色。
- 返回：像一个角落的皮影剪影，点击时剪影亮起。

**边界感消除**：
- ❌ 去除：左侧面板矩形框、右侧面板矩形框、线索卡片矩形、筛选标签矩形。
- ✅ 保留：旧纸本身（作为信息载体）、旧纸上的文字、暗红印章（已使用线索）。
- ✅ 新增：旧纸可以有轻微旋转（±3°），像真的散落在桌上。hover 时纸浮起，阴影加深。

### 5. 角色档案：摊开的侦探笔记与旧照片

**布局（16:9）**：
- 背景：暗场桌面，像侦探在桌上摊开一本笔记和几张旧照片。
- 左侧：角色 2D 立绘——像一张竖立的旧照片，照片边缘破损，没有矩形框。
- 中间：侦探笔记——像摊开的笔记本，纸页是不规则形状，上面手写文字。表面身份、行当、可疑点像笔记中的条目。
- 右侧：隐藏画像——像另一张被黑雾覆盖的照片，放在笔记旁边。黑雾是 2D 手绘墨雾，覆盖在照片上方。
- 关联线索：像贴在笔记边缘的小纸条，每张纸条是不规则形状，有暗红印章。

**边界感消除**：
- ❌ 去除：角色信息矩形面板、档案卡片矩形、隐藏画像矩形框。
- ✅ 保留：旧照片本身（作为信息载体）、笔记纸页、纸条上的文字、暗红印章。
- ✅ 新增：笔记页可以有轻微卷曲（用 2D 插画表现），像真的摊开在桌上。

### 6. 夜晚观皮界面：直接面对黑雾中的画像

**布局（16:9）**：
- 背景：极暗烧毁剧院/戏台，背景装饰 opacity ≤ 10%，几乎纯黑。
- 中央 45%：被黑雾覆盖的隐藏画像——像挂在墙上的一张旧照片，照片本身没有明显边框，只有极淡的暗铜线稿（opacity 0.3）。黑雾是 2D 手绘墨雾，多个不规则黑色圆形叠加，覆盖在照片上方。玩家感觉像真的在看一张被黑雾遮住的照片。
- 左侧：角色信息——像墨迹浮在暗场中，只有文字，没有底板。角色名、行当、可疑点像用墨写在黑雾边缘。
- 右侧：可用线索——像漂浮的旧纸签/墨迹，每个线索是不规则形状，没有矩形卡片。
- 底部：操作提示——像暗红烛光/墨迹字，极小，不抢中央画像焦点。

**边界感消除**：
- ❌ 去除：所有矩形面板、画像矩形框、线索矩形卡片、档案底板。
- ✅ 保留：画像本身（中央焦点）、黑雾（2D墨雾）、线索文字（可点击）、角色名（可识别）。
- ✅ 核心：观皮界面没有“打开了一个面板”的感觉，玩家感觉像真的站在黑雾中的画像前。

### 7. 黑雾揭露反馈：从场景中直接显现

**反馈设计（无边界版）**：
- **线索选中**：线索像被墨笔点中，暗红线条从线索位置延伸到画像区域（手绘线，不是几何直线）。
- **正确匹配**：黑雾从线索延伸点向外退散，像墨水被冲开。没有“弹窗”或“通知”，而是画像本身局部变亮，新信息像从画像中浮现。
- **错误匹配**：线索像被黑墨吞回，纸签变暗并轻微收缩（scale 0.95），没有弹窗，只有纸签本身的反馈。
- **关键揭露**：画面没有弹窗，而是中央画像局部压暗四周，只留白脸/伤痕/裂纹被照亮。新文字像从画像中浮现，不是从面板中弹出。
- **新目标**：像一张旧纸条从画面底部自然滑入，贴在场景中，2 秒后融入黑雾。

### 8. 暂停 / 设置：暗场中的微弱烛光

**布局（16:9）**：
- 背景：全屏半透明黑底（`#0a0808`，opacity 0.8），保留当前场景轮廓，但压暗。
- 设置选项：像暗场中的微弱烛光/墨迹字，垂直排列在画面中央。每个选项没有矩形底板，只有文字和极淡的暗铜线稿（opacity 0.3）。
- Slider：像旧纸上的墨迹线，滑块是暗铜小点，不是现代矩形滑块。
- Toggle：像旧纸上的勾选框，手绘对勾，无矩形边框。
- 按钮：像暗场中的墨迹字，hover 时字亮（暗金），没有矩形底。

**边界感消除**：
- ❌ 去除：设置面板矩形框、Slider 矩形轨道、Toggle 矩形框、按钮矩形底。
- ✅ 保留：文字本身、Slider 墨迹线、Toggle 手绘对勾、按钮文字 hover 态。

## 五、页面流程（V3 无边界版）

```
[主菜单：空戏台]
   │ 点击悬挂的旧戏票
   ▼
[白天探索：场景中的墨迹]
   │ 点击场景物件（虚线描边）→ 墨迹浮现物件名称
   │ 点击 NPC → 对话字幕浮现
   ▼
[对话：戏台字幕与漂浮纸条]
   │ 推进对话 → 字幕逐字显现
   │ 选择选项 → 旧纸签浮现
   ▼
[线索背包：散落的旧纸]
   │ 点击旧纸 → 纸张浮起，展开详情
   ▼
[角色档案：摊开的笔记与照片]
   │ 查看笔记 → 纸页翻动
   ▼
[观皮：直接面对黑雾画像]
   │ 点击旧纸签 → 墨迹延伸到画像
   │ 正确匹配 → 黑雾退散，画像局部显现
   │ 错误匹配 → 纸签被墨吞回
   ▼
[新目标：旧纸条滑入场景]
```

## 六、Canvas 分层与 Sort Order（V3 调整）

V3 保持 7 层结构，但调整各层的使用方式：**更多层使用透明度、渐变、无边界元素**。

| 层级 | Canvas 名称 | Sort Order | V3 调整说明 | 常驻/按需 | 阻断输入 | 背景可见 |
|------|------------|------------|------------|-----------|----------|----------|
| 0 | BackgroundCanvas | 0 | 全屏 2D 场景插画（戏台、剧院、暗场），无 UI 元素 | 常驻 | 否 | 是 |
| 10 | WorldHUDCanvas | 10 | 半透明墨迹文字、虚线描边、角落旧戏票，无矩形底板 | 常驻 | 否 | 是 |
| 20 | DialogueCanvas | 20 | 暗场渐变字幕、漂浮旧纸签、悬挂戏牌，无矩形对话框 | 按需 | 是 | 是（半透明） |
| 30 | NormalPanelCanvas | 30 | 散落的旧纸、摊开的笔记、不规则照片，无矩形面板 | 按需 | 是 | 是（半透明） |
| 40 | PopupCanvas | 40 | 墨迹浮现提示、旧纸条滑入，无弹窗矩形 | 按需 | 部分 | 是 |
| 50 | RevealCanvas | 50 | 黑雾退散、画像局部高亮、四周压暗，无面板 | 按需 | 是 | 否（覆盖） |
| 60 | SystemCanvas | 60 | 全屏半透明黑底 + 墨迹文字/烛光，无设置面板 | 按需 | 是 | 是（半透明黑底） |

### V3 关键变化
- **BackgroundCanvas（0）**：现在承载全屏 2D 场景插画，不是单纯的“背景图”。主菜单时就是空戏台，探索时就是场景，观皮时就是烧毁剧院。
- **WorldHUDCanvas（10）**：所有 HUD 元素去除矩形底板，改用半透明文字 + 暗场渐变。
- **DialogueCanvas（20）**：去除旧纸卷对话框，改用暗场渐变字幕 + 漂浮旧纸签。
- **NormalPanelCanvas（30）**：去除左右面板，改用散落的旧纸和摊开的笔记。
- **PopupCanvas（40）**：去除弹窗矩形，改用墨迹浮现和旧纸条滑入。
- **SystemCanvas（60）**：去除设置面板矩形，改用全屏半透明黑底 + 墨迹文字。

## 七、UI 管理结构与模块职责（V3 调整）

### 核心模块调整

| 脚本 | V3 职责变化 | 说明 |
|------|------------|------|
| **UIManager** | 增加“场景状态”管理，不仅管理面板，还管理场景切换（白天/夜晚/观皮） | 打开面板时，背景场景仍然可见，只是压暗或模糊 |

## 九、Unity UGUI 实现建议（V3 无边界）

### 1. Canvas 与透明度

- **BackgroundCanvas**：使用 `Screen Space - Overlay`，Sort Order 0。背景图是全屏 2D 插画（1920×1080 PNG）。
- **其他层**：使用 `Screen Space - Overlay`，Sort Order 10–60。所有 UI 元素的 `Image` 组件使用半透明颜色（`color.a` 0.4–0.9）。
- **关键**：使用 `CanvasGroup` 控制整层透明度，而不是单个 `Image`。

### 2. 无边框实现

- **去除 `Image` 的默认矩形**：使用不规则 Sprite（2D 插画师提供），`Image.type = Simple`。
- **边框处理**：不要给 `Image` 加 `Outline` 组件，而是使用极淡线稿 Sprite（单独一层，opacity 0.2–0.4）。
- **背景处理**：不要使用 `Image` 的实色背景，而是使用渐变 Sprite（从 `#0a0808` opacity 0.8 渐变到透明）。

### 3. 遮罩与黑雾

- **黑雾遮罩**：使用多个 `Image`（圆形 Sprite）叠加，每个 `Image.color = #0a0808`，`alpha` 0.85–0.95。
- **黑雾退散**：使用 `CanvasGroup.alpha` 或 `Image.color.a` 渐变，从 0.95 到 0.2，持续 500–800ms。
- **关键揭露压暗**：在 RevealCanvas 上添加全屏 `Image`（`#0a0808`，alpha 0.7），中央用遮罩挖洞（或用一个透明圆形区域），只保留中央画像区域可见。

### 4. 动效与无边界感

- **浮现效果**：使用 `CanvasGroup.alpha` + `RectTransform.anchoredPosition`。
  - 文字浮现：`alpha` 0→1（300ms），`y` 从 +10 到 0（轻微上浮感）。
  - 旧纸浮现：`alpha` 0→1（400ms），`scale` 从 0.95 到 1（轻微放大感），`rotation` 从 ±5° 到 0（像纸被铺平）。
- **黑雾退散**：`CanvasGroup.alpha` 从 0.95 到 0.2（800ms），配合 `scale` 从 1 到 1.1（向外扩散感）。
- **错误墨吞回**：`Image.color.a` 从 1 到 0.3（200ms），`scale` 从 1 到 0.95（收缩感），然后恢复。
- **关键揭露停顿**：全屏黑底 `alpha` 从 0 到 0.7（500ms），停顿 0.5 秒，中央画像区域 `alpha` 保持，然后黑底 `alpha` 从 0.7 到 0.3（800ms），新文字浮现。

### 5. 交互反馈

- **Hover**：暗金描边（`Image.color` 从 `#5a3a2a` 到 `#c9a96e`，100ms），或元素轻微上浮（`y` 从 0 到 -3px）。
- **选中**：元素 `scale` 从 1 到 1.02（轻微放大），暗金描边亮起。
- **可点击区域**：使用 `Image` 的 `raycastTarget = true`，但 `Image.color.a` 可以很低（0.1），玩家看不到明显的点击区域，但鼠标悬停时有反馈。
- **拖拽**：观皮时拖拽线索到画像，线索像被墨笔拖曳，留下淡淡墨迹轨迹（用 `TrailRenderer` 或 `LineRenderer` 模拟）。

### 6. 场景融合技术

- **主菜单**：背景图是全屏戏台插画，菜单项是悬挂在背景上的 Sprite（用锚点固定在背景特定位置）。
- **对话**：背景是场景图（半暗），角色立绘是左侧 Sprite，对话文字是底部半透明渐变文字。
- **线索背包**：背景是侦探房间插画，线索是散落在背景上的 Sprite，点击时放大到中央。
- **观皮**：背景是烧毁剧院插画，画像是中央 Sprite，黑雾是覆盖在画像上的多个 Sprite。

## 十一、关键设计原则总结（V3）

> **UI 不是放在画面上的面板，而是从《画皮》世界中浮现的信息。**

1. **场景即界面**：主菜单就是戏台，对话就是字幕，线索就是散落的旧纸，观皮就是黑雾中的画像。
2. **信息无框**：文字直接出现在场景中，背后只有极暗渐变，没有矩形底板。
3. **元素无形**：按钮像旧戏票，选项像旧纸签，线索像旧照片，所有元素都是不规则形状。
4. **黑雾即遮罩**：黑雾不是“UI特效”，而是场景本身的一部分，2D 手绘墨雾覆盖在画像上。
5. **动效即氛围**：浮现、退散、墨吞回，所有动效都像是水墨和旧纸在变化，不是数字面板的动画。
6. **可读性不妥协**：即使无边界，文字背后仍有暗底渐变，对比度 ≥ 4.5:1，字号不小于 12px。
